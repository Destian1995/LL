
# Легенды Лэрдона (Lerdon Legends) - Полная документация

Стратегическая игра в реальном времени с элементами экономики, военной стратегии и политики. Игрок управляет княжеством, конкурируя с другими фракциями, контролируемыми продвинутым ИИ.

---

## 🎮 Описание проекта

**Легенды Лэрдона** — это пошаговая стратегическая игра, где:
- 🏰 Игрок управляет княжеством с городами и ресурсами
- 💰 Развиваете экономику: строительство зданий, торговля, налоги
- ⚔️ Формируете армию и ведете войны с другими фракциями
- 🤝 Заключаете политические союзы и договоры
- 🎭 Сталкиваетесь со случайными событиями
- 🤖 Конкурируете с интеллектуальными противниками

**Платформа:** Python 3.11 + Kivy + KivyMD (мобильные и десктопные приложения)

## 🎯 Основные компоненты

### 1. **main.py** - Точка входа и главное меню
```
Функции:
- Инициализация приложения Kivy
- Отображение главного меню
- Выбор фракции
- Загрузка ранее сохраненных игр
- Инициализация базы данных и асхет
```

### 2. **game_process.py** - Основной игровой цикл
```
Содержит:
- GameScreen: основной класс экрана игры
- Управление ходами (игрок → ИИ фракция 1 → ИИ фракция 2 → ...)
- Обработка действий игрока через UI кнопки
- Координация между модулями (economic, army, politic)
- Отрисовка карты и интерфейса
- Обновление состояния игры в реальном времени

Цикл:
1. Игрок нажимает кнопку действия
2. Модуль обновляет БД и состояние
3. Передача хода ИИ фракциям
4. Каждая ИИ фракция выполняет свой ход
5. Обновление интерфейса
6. Передача хода обратно игроку
```

### 3. **economic.py** -模块 экономики
```
Класс: EconomicCash
Функции:
- Строительство зданий (жилые, оборонные, производство)
- Управление ресурсами (деньги, население)
- Торговля с другими фракциями
- Взимание налогов
- Расчет производства и дохода
- Обновление состояния зданий

Переменные:
- buildings: {building_type: count}
- money: текущие деньги фракции
- population: население в городе
- resources: производимые ресурсы

Методы:
- build_building(building_type, count)
- calculate_income()
- pay_taxes()
- trade_with(faction, resource, amount)
```

### 4. **army.py** - Модуль армии
```
Класс: ArmyCash
Функции:
- Создание и управление войсками
- Командование боем (нажатие кнопки "Атака")
- Рассчет потерь и урона
- Управление боевым духом армии
- Прием подкреплений

Боевые единицы:
- Пехота (низкие затраты, большое количество)
- Кавалерия (мобильность)
- Лучники (урон издалека)
- Осадные машины (против стен)

Механика боя:
- Сравнение силы армий
- Расчет потерь на основе численности и оборудования
- Учет укреплений города
- Моральный фактор
```

### 5. **fight.py** - Механика боя
```
Функции:
- calculate_battle_result(attacker_army, defender_army, defender_fortification)
- Определение победителя
- Расчет потерь
- Обновление состояния армий после боя

Учитывает:
- Численность войск
- Качество вооружения
- Укрепления защищаемого города
- Боевой дух
- Тактика командира
```

### 6. **politic.py** - Политика и дипломатия
```
Функции:
- Заключение союзов
- Объявление войны
- Торговые соглашения
- Обмен данными
- Отправка посольств
- Система рейтинга отношений между фракциями

Отношения:
- Враждебные (-100)
- Нейтральные (0)
- Союзные (+100)
```

### 7. **ii.py** - ИИ контроллер
```
Класс: AIController
Функции:
- Анализ текущего состояния ресурсов

- Расчет торговых сделок
- Управление дипломатией

Стратегии ИИ:
- Агрессивная (расширение территорий)
- Экономическая (развитие производства)
- Защитная (укрепление)
- Дипломатическая (союзы и торговля)
```

## 📊 Модель данных (SQLite schema)

```sql
-- Фракции
CREATE TABLE factions (
    id INTEGER PRIMARY KEY,
    name TEXT UNIQUE,
    color TEXT,
    treasury REAL,
    population INTEGER,
    research_progress INTEGER,
    status TEXT
);

-- Города
CREATE TABLE cities (
    id INTEGER PRIMARY KEY,
    name TEXT,
    faction TEXT,
    x INTEGER, y INTEGER,    # координаты на карте
    population INTEGER,
    fortification INTEGER,
    resources TEXT,          # JSON с ресурсами
    FOREIGN KEY (faction) REFERENCES factions(name)
);

-- Здания
CREATE TABLE buildings (
    id INTEGER PRIMARY KEY,
    city_name TEXT,
    faction TEXT,
    building_type TEXT,
    count INTEGER,
    FOREIGN KEY (city_name) REFERENCES cities(name)
);

-- Армии
CREATE TABLE armies (
    id INTEGER PRIMARY KEY,
    faction TEXT,
    city_name TEXT,
    unit_type TEXT,          # пехота, кавалерия, лучники
    count INTEGER,
    morale INTEGER            # мораль 0-100
);

-- Дипломатические отношения
CREATE TABLE diplomacy (
    id INTEGER PRIMARY KEY,
    faction1 TEXT,
    faction2 TEXT,
    relationship_type TEXT,   # союз, война, нейтралитет
    level INTEGER,            # -100 до +100
    treaty_expires INTEGER
);

-- События
CREATE TABLE events (
    id INTEGER PRIMARY KEY,
    faction TEXT,
    event_type TEXT,
    description TEXT,
    timestamp INTEGER
);
```

## 🎮 Игровой цикл (Game Loop)

```
1. ИНИЦИАЛИЗАЦИЯ
   ├── Загрузка БД
   ├── Инициализация фракций
   ├── Генерация карты
   └── Создание ИИ контроллеров

2. ЦИКЛ ИГРЫ (повторяется каждый ход)
   ├── ХОД ИГРОКА
   │   ├── Отрисовка интерфейса
   │   ├── Ожидание действия (эконом, армия, политика)
   │   └── Обновление состояния в БД
   │
   ├── ХОДЫ ИИ (для каждой фракции)
   │   ├── Анализ состояния (AIController.analyze())
   │   ├── Принятие решения (AIController.decide_action())
   │   ├── Выполнение действия
   │   └── Обновление БД
   │
   ├── СЛУЧАЙНЫЕ СОБЫТИЯ
   │   ├── Генерация EventManager.generate_event()
   │   └── Применение эффектов
   │
   ├── СМЕНА СЕЗОНОВ (каждый 4-й ход)
   │   ├── SeasonManager.next_season()
   │   └── Обновление производства
   │
   └── ОБНОВЛЕНИЕ ИНТЕРФЕЙСА
       ├── Отрисовка карты
       ├── Показ уведомлений
       └── Синхронизация видео с БД

3. КОНЕЦ ИГРЫ
   ├── Проверка условий победы
   ├── Сохранение результатов (ResultsGame)
   └── Отображение итогам экрана
```

## 🤖 Стратегия ИИ (ii.py)

```python
AIController.execute_turn():
    1. Анализ ресурсов
       - Текущее золото
       - Численность войск
       - Состояние зданий
       - Уровень угрозы
    
    2. Принятие решений
       - Если мало денег → строить фермы
       - Если мало войск → набирать
       - Если враг слаб → атаковать
       - Если врагов 2 → искать союзника
    
    3. Выполнение действия
       - Вызов economic.py
       - Вызов army.py
       - Вызов politic.py
    
    4. Обновление БД
       - Сохранение состояния
       - Логирование действий
```
